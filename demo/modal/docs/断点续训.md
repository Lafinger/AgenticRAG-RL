# Modal 断点续训

本流程用于切换 Modal 账号或 workspace 后，从旧账号 checkpoint 继续 2xH100 正式 GRPO 训练。

默认正式输出目录不变：

```text
/vol/outputs/grpo_tool_agent_react_v4_h100x2
```

对应 Modal Volume 路径：

```text
/grpo_tool_agent_react_v4_h100x2
```

代码里已设置：

```text
trainer.resume_mode=auto
```

只要新账号的 `agentic-rag-rl-outputs` Volume 里存在 `latest_checkpointed_iteration.txt` 和对应 `global_step_<STEP>` 目录，重新启动 `train` 会自动续训。

## 1. 切换账号前：下载旧账号最新 checkpoint

确认当前 Modal profile 仍是旧账号，然后执行：

```powershell
cd E:\AI\AgenticRAG-RL

$env:PYTHONUTF8 = "1"
$env:PYTHONIOENCODING = "utf-8"
[Console]::OutputEncoding = [System.Text.UTF8Encoding]::new()

.\demo\.venv\Scripts\python.exe .\demo\modal\download_latest_checkpoint.py
```

脚本会自动读取：

```text
agentic-rag-rl-outputs:/grpo_tool_agent_react_v4_h100x2/latest_checkpointed_iteration.txt
```

并下载到：

```text
demo/training/outputs/modal_resume_global_step_<STEP>
```

如果只想先下载续训必需状态、暂时跳过 HuggingFace 导出权重：

```powershell
.\demo\.venv\Scripts\python.exe .\demo\modal\download_latest_checkpoint.py --skip-hf-weights
```

只预览不下载：

```powershell
.\demo\.venv\Scripts\python.exe .\demo\modal\download_latest_checkpoint.py --dry-run
```

当前已下载断点：

```text
demo/training/outputs/modal_resume_global_step_160
latest_checkpointed_iteration.txt = 160
```

已知状态：

- 旧账号 `global_step_160` 的 `actor/optim_world_size_2_rank_1.pt` 已损坏，直接按默认 `load_contents=[model,optimizer,extra]` 恢复会报 `PytorchStreamReader failed reading file ... invalid header or archive is corrupted`。
- 新账号 workspace `dnywijrooaoanxbdhsia` 已完成 data/models/outputs/secrets 准备，远端 `check_inputs` 已通过。
- 2026-05-22 已用 `actor_rollout_ref.actor.checkpoint.load_contents=[model,extra]` 从 `global_step_160` 预检跑到 `global_step_161`，并已停止预检 App。
- 当前远端最新断点为 `agentic-rag-rl-outputs:/grpo_tool_agent_react_v4_h100x2/global_step_161`，`latest_checkpointed_iteration.txt = 161`。

## 2. 切换账号后：准备新 workspace 并上传断点

先切换 Modal 账号或 workspace：

```powershell
.\demo\.venv\Scripts\modal.exe token new --profile <新profile名> --activate
```

或使用已有 token：

```powershell
.\demo\.venv\Scripts\modal.exe token set `
  --token-id <新账号TOKEN_ID> `
  --token-secret <新账号TOKEN_SECRET> `
  --profile <新profile名> `
  --activate
```

确认当前账号：

```powershell
.\demo\.venv\Scripts\modal.exe profile current
.\demo\.venv\Scripts\modal.exe token info
```

准备 Volume、上传数据/模型、上传 checkpoint，并运行 `check_inputs`：

```powershell
.\demo\.venv\Scripts\python.exe .\demo\modal\resume_after_account_switch.py --create-placeholder-secret
```

如果已经手动创建了真实 Secret，可以不传 `--create-placeholder-secret`。

如果已经上传过 data/models，只补传 checkpoint：

```powershell
.\demo\.venv\Scripts\python.exe .\demo\modal\resume_after_account_switch.py --skip-assets
```

只预览不创建、不上传、不运行远端检查：

```powershell
.\demo\.venv\Scripts\python.exe .\demo\modal\resume_after_account_switch.py --dry-run
```

## 3. 显式启动续训

默认脚本不会启动训练，避免误触 2xH100 费用。

准备完成后，确认没有运行中的训练任务：

```powershell
.\demo\.venv\Scripts\modal.exe app list
```

启动续训：

```powershell
.\demo\.venv\Scripts\python.exe .\demo\modal\resume_after_account_switch.py `
  --skip-assets `
  --skip-checkpoint-upload `
  --start-training `
  --train-extra-arg "actor_rollout_ref.actor.checkpoint.load_contents=[model,extra]"
```

这里必须保留 `load_contents=[model,extra]`，原因是 `global_step_160` 的 rank1 optimizer checkpoint 已损坏。该参数会加载模型、extra state、rng 和 lr_scheduler，但重新初始化 optimizer。等后续确认新的 checkpoint 可完整加载后，再评估是否恢复默认 optimizer 加载。

如果只想做一次短预检，确认能恢复并进入训练循环后自动停在下一步：

```powershell
.\demo\.venv\Scripts\python.exe .\demo\modal\resume_after_account_switch.py `
  --skip-assets `
  --skip-checkpoint-upload `
  --start-training `
  --train-extra-arg "actor_rollout_ref.actor.checkpoint.load_contents=[model,extra]" `
  --train-extra-arg "trainer.total_training_steps=161"
```

脚本会先检查 `modal app list --json`，发现已有运行中的 `agentic-rag-rl-grpo` 时会拒绝启动。

启动成功后，用脚本输出的 App ID 跟踪日志：

```powershell
.\demo\.venv\Scripts\modal.exe app logs <APP_ID> -f --timestamps
```

日志里重点确认：

```text
retrieval server ready
/search HTTP/1.1" 200 OK
Training Progress
local_global_step_folder: /vol/outputs/grpo_tool_agent_react_v4_h100x2/global_step_*
```

如果已经确认进入训练循环，需要手动停止预检或误启动的 App：

```powershell
.\demo\.venv\Scripts\modal.exe app stop -y <APP_ID>
.\demo\.venv\Scripts\modal.exe app list --json
```

停止后确认对应 App 为 `State = stopped` 且 `Tasks = 0`。

## 常见问题

- `workspace billing cycle spend limit reached`：这是 Modal workspace 预算上限。需要提高 workspace budget，或切到新 workspace 后按本文迁移 checkpoint。
- Windows 目录下载 `Permission denied`：使用 `download_latest_checkpoint.py`，它逐文件下载，规避 Modal CLI 直接下载目录的问题。
- 大文件下载长时间无输出：Modal CLI 下载大文件时可能不显示进度。另开 PowerShell 查看本地文件大小是否增长。
- 不允许并发写同一目录：同一时间只能有一个训练任务写 `/vol/outputs/grpo_tool_agent_react_v4_h100x2`。
- 续训 OOM：不要反复重试，下一轮优先降低 `data.train_batch_size`、`actor_rollout_ref.actor.ppo_mini_batch_size` 和 `actor_rollout_ref.rollout.gpu_memory_utilization`。
- `PytorchStreamReader failed reading file ... invalid header or archive is corrupted`：通常是 checkpoint 文件损坏。本次已确认 `global_step_160/actor/optim_world_size_2_rank_1.pt` 损坏，正式续训需追加 `--train-extra-arg "actor_rollout_ref.actor.checkpoint.load_contents=[model,extra]"` 跳过 optimizer 加载。
