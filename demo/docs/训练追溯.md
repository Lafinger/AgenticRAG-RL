# SFT 训练样本追溯

本文说明如何精确追溯 SFT/QLoRA 训练中某个 `Gradient Norm` 突刺对应的训练样本。先给追溯方法，后面再说明实现细节。

## 追溯方法

要精确追溯某个 optimizer step 使用了哪些样本，需要使用本工程的可追溯训练入口：

```text
scripts\train_sft_unsloth_trace.py
```

它会在训练时写出：

```text
step_sample_trace.jsonl
trace_metrics.jsonl
trace_training_state.json
```

其中 `step_sample_trace.jsonl` 是核心文件，记录每个 optimizer step 对应的全部 micro-batch 样本。

当前可追溯训练和标准 SFT 使用同一条 tool-aware 渲染路径：每条样本读取 `messages` 和 record 级 `tools`，通过 `tokenizer.apply_chat_template(..., tools=tools)` 生成训练文本，再用 assistant-only label mask 只监督 assistant token。`source_line` 仍对应输入 SFT JSONL 的行号，不对应渲染后的文本行号。

### 1. 启动可追溯训练

在 PowerShell 中执行：

```powershell
Set-Location E:\AI\AgenticRAG-RL\demo

uv run --no-sync python .\scripts\train_sft_unsloth_trace.py `
  --config .\training\unsloth_sft.yaml `
  --data-path .\data\novel_eval\sft_zh_unsloth\train_cli.jsonl `
  --eval-data-path .\data\novel_eval\sft_zh_unsloth\eval.jsonl `
  --output-dir .\training\outputs\unsloth_sft_qwen3_4b_lora_trace
```

训练超参统一从 `training\unsloth_sft.yaml` 读取：

```text
batch_size = 2
gradient_accumulation_steps = 8
learning_rate = 1e-4
eval_steps = 105
max_grad_norm = 1.0
max_seq_length = 2048
train_cli.jsonl = 5800 条
eval.jsonl = 200 条
```

因此 3 个 epoch 约为：

```text
ceil(5800 / (2 * 8)) * 3 = 1089 steps
```

训练输出目录：

```text
training\outputs\unsloth_sft_qwen3_4b_lora_trace
```

关键追溯文件：

```text
training\outputs\unsloth_sft_qwen3_4b_lora_trace\step_sample_trace.jsonl
training\outputs\unsloth_sft_qwen3_4b_lora_trace\trace_metrics.jsonl
```

### 2. 查找 Gradient Norm 最大的 step

如果你不知道突刺在哪个 step，先查最大 `grad_norm`：

```powershell
uv run python .\scripts\inspect_sft_trace.py `
  --trace .\training\outputs\unsloth_sft_qwen3_4b_lora_trace\step_sample_trace.jsonl `
  --top-grad-norm 5
```

这会输出 grad norm 最大的 5 个 step，并列出每个 step 对应的样本。

也可以导出报告：

```powershell
uv run python .\scripts\inspect_sft_trace.py `
  --trace .\training\outputs\unsloth_sft_qwen3_4b_lora_trace\step_sample_trace.jsonl `
  --top-grad-norm 5 `
  --output .\results\sft_trace_top_grad_norm.json
```

### 3. 按指定 step 追溯样本

如果你已经知道突刺发生在 step 199：

```powershell
uv run python .\scripts\inspect_sft_trace.py `
  --trace .\training\outputs\unsloth_sft_qwen3_4b_lora_trace\step_sample_trace.jsonl `
  --step 199
```

导出为 JSON：

```powershell
uv run python .\scripts\inspect_sft_trace.py `
  --trace .\training\outputs\unsloth_sft_qwen3_4b_lora_trace\step_sample_trace.jsonl `
  --step 199 `
  --output .\results\sft_trace_step_199.json
```

输出中重点看：

```text
step
loss
grad_norm
eval_loss
samples[].source_line
samples[].sample_id
samples[].question
samples[].answer
samples[].hop_count
samples[].gold_chunks
samples[].token_length
samples[].truncated
```

### 4. 定位源样本

`source_line` 对应训练 JSONL 文件中的行号：

```text
data\novel_eval\sft_zh_unsloth\train_cli.jsonl
```

例如输出里出现：

```json
{
  "source_line": 1587,
  "question": "...",
  "hop_count": 3
}
```

说明该 step 使用了 `train_cli.jsonl` 第 1587 行样本。可以用 PowerShell 查看：

```powershell
$line = 1587
Get-Content .\data\novel_eval\sft_zh_unsloth\train_cli.jsonl -Encoding UTF8 |
  Select-Object -Index ($line - 1)
```

### 5. 如何判断是不是异常样本

追溯到样本后，优先看这些字段：

| 字段 | 判断方式 |
| --- | --- |
| `token_length` | 是否明显长于其他样本 |
| `truncated` | 是否被截断，正常应为 `false` |
| `hop_count` | 3-hop 样本更容易带来大梯度 |
| `question` | 是否存在异常长问题或不自然合成问题 |
| `gold_chunks` | 是否跨小说、跨章节或证据链异常 |
| `messages/tools` 原文 | 是否存在格式损坏、缺少 `tools`、tool role 被改写、空答案、工具返回异常 |

当前 canonical ReAct 数据最长约 1978 tokens，`max_seq_length=2048` 可以覆盖；如果 trace 中出现 `truncated=true`，说明后续数据变长或工具 schema 变大，需要重新运行 `scripts\calc_sample_lengths.py`。

## 输出文件说明

### step_sample_trace.jsonl

每行对应一个 optimizer step。

结构示例：

```json
{
  "step": 199,
  "epoch": 0.57,
  "loss": 0.4491,
  "learning_rate": 0.00008145,
  "grad_norm": 19.3473,
  "eval_loss": null,
  "micro_batches": [
    {
      "micro_batch_index": 789,
      "epoch_index": 1,
      "epoch_progress": 0.56,
      "items": [
        {
          "sample_id": "sft_001585",
          "source_line": 1585,
          "token_length": 1507,
          "truncated": false,
          "question": "...",
          "answer": "...",
          "hop_count": 2,
          "gold_chunks": ["yttlj_1238", "yttlj_1237"]
        }
      ]
    }
  ],
  "sample_ids": ["sft_001585"],
  "source_lines": [1585]
}
```

如果：

```text
per_device_train_batch_size = 2
gradient_accumulation_steps = 8
```

那么每个 step 通常会记录：

```text
8 个 micro_batch
16 条样本
```

### trace_metrics.jsonl

每行对应一个 optimizer step 的简化指标：

```json
{
  "step": 199,
  "epoch": 0.57,
  "loss": 0.4491,
  "learning_rate": 0.00008145,
  "grad_norm": 19.3473,
  "eval_loss": null,
  "sample_count": 8,
  "source_lines": [1585, 1586, 1587, 1588, 1589, 1590, 1591, 1592],
  "sample_ids": ["sft_001585", "sft_001586"]
}
```

该文件适合快速画图或搜索最大 `grad_norm`。

### trace_training_state.json

记录训练最新状态：

```text
global_step
total_steps
last_loss
last_grad_norm
last_learning_rate
last_eval_loss
trace_output
metrics_output
```

中断训练时可以先看这个文件确认当前进度。

## 实现细节

### 新训练入口如何做到精确

`scripts\train_sft_unsloth_trace.py` 在项目脚本里显式控制训练循环：

1. 读取 `train_cli.jsonl`。
2. 为每条样本生成稳定的 `sample_id` 和 `source_line`。
3. 使用 tokenizer 的 `apply_chat_template` 渲染训练文本。
4. 用自定义 collator 把 `sample_id/source_line/question/answer/hop_count/gold_chunks/token_length` 带进每个 batch。
5. 每个 micro-batch 先放入 `trace_buffer`。
6. 到达 `gradient_accumulation_steps` 后执行一次 optimizer step。
7. 写出该 optimizer step 对应的所有 micro-batch 样本。

因此：

```text
step_sample_trace.jsonl 第 N 行 = optimizer step N 的真实样本集合
```

不需要猜测 shuffle 顺序。

### 训练行为和普通脚本的关系

可追溯训练脚本复用同一套核心配置：

```text
model_name_or_path
max_seq_length
load_in_4bit
lora_rank
lora_alpha
lora_dropout
lora_target_modules
learning_rate
num_train_epochs
batch_size
gradient_accumulation_steps
warmup_steps
seed
```

但它不使用 TRL `SFTTrainer`，而是使用显式 PyTorch training loop。这样可以保证样本追溯精确。训练目标仍是 causal LM SFT：输入为完整 chat template 文本，labels 为同一序列，padding 位置使用 `-100` 忽略。

### 为什么默认输出到独立目录

如果不传 `--output-dir`，脚本会把普通 SFT 配置中的输出目录加 `_trace` 后缀，避免覆盖普通 LoRA 输出。

建议显式指定：

```text
training\outputs\unsloth_sft_qwen3_4b_lora_trace
```

### 是否可以关闭 shuffle

可以用：

```powershell
--no-shuffle
```

关闭后，step 与源文件行号可以直接换算，便于 debug。但正式训练不建议关闭 shuffle，因为固定顺序可能影响训练质量。可追溯脚本在 shuffle 开启时也能精确记录真实样本，因此一般不需要关闭。

## 常见追溯结论

### 单个 step 突刺

如果只有单点 `grad_norm` 突刺，但：

```text
loss 没有持续上升
eval_loss 没有持续上升
后续 grad_norm 立即恢复
```

通常不需要处理。追溯样本只用于确认是否存在明显脏数据。

### 同一类样本反复导致突刺

如果多个高 `grad_norm` step 都包含相似样本，例如：

```text
3-hop
超长 tool_response
跨小说 gold_chunks
答案别名异常
```

可以考虑清洗对应样本或降低该类样本占比。

### 频繁突刺并伴随 eval loss 上升

优先调整：

```text
learning_rate: 1e-4 -> 5e-5
max_grad_norm: 1.0
Context Length: 保持覆盖样本即可，不要无谓增大
检查 trace 中高 grad_norm step 的样本格式
```
