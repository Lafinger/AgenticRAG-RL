# 架构说明

## 数据链路

默认链路：

`data/original_data/*.txt -> novel/corpus.jsonl -> FAISS/BM25/KG indexes -> seeds -> multihop qa -> oracle traces -> canonical ReAct SFT(messages/tools/tool role) -> Unsloth SFT JSONL -> grpo parquet`

默认产物：

- `data/novel/corpus.jsonl`
- `data/novel/indexes/`
- `data/novel_eval/seeds.jsonl`
- `data/novel_eval/qa_pairs.jsonl`
- `data/novel_eval/traces_oracle_zh.jsonl`
- `data/novel_eval/sft/`
- `data/novel_eval/sft_zh_unsloth/`
- `data/novel_eval/grpo_agentic_train.parquet`
- `data/novel_eval/grpo_agentic_val.parquet`

## 训练链路

- SFT：`scripts/train_sft_unsloth.py` + `training/unsloth_sft.yaml`
- LoRA 合并：`scripts/export_unsloth_lora.py`
- GRPO / RL：`scripts/train_grpo_unsloth.py` + `training/unsloth_grpo.yaml`
- Reward：`training/reward_agentic_rag.py`
- Retrieval server：`training/tools/retrieval_server.py`

## SFT / Agent Loop / GRPO 协议

主 SFT 基线采用 Qwen3 tool-aware canonical ReAct 协议：

- `data/novel_eval/sft/react.jsonl`、`sharegpt.jsonl`、`messages.jsonl` 每行都包含 `messages`、`tools`、`metadata`。
- `messages` 保留 `tool` role；tool content 只保存原始检索文本，不提前手写 `<tool_response>`。
- assistant 工具轮使用 `<think>...</think>\n<tool_call>\n{...}\n</tool_call>`，最终答案使用 `<answer>...</answer>`。
- 训练、长度统计和 HF Agent loop 都传入 `tokenizer.apply_chat_template(..., tools=tools)`，由 Qwen3 模板注入 `# Tools`、`<tools>` 和 `<tool_response>`。
- GRPO parquet 行也写入 `tools` 字段，后续 rollout 不再单独维护另一份工具定义。
- direct-answer 和 agentic-stop 数据只作为补充诊断路线，不再作为主 SFT 基线。

## 本机与远端职责

- 本机 Windows：小说文本解析、数据处理、SFT 数据转换、CPU retrieval server、smoke evaluation、小样本训练验证。
- 外部 GPU 环境：Unsloth SFT、Unsloth GRPO/RL、长时间训练和大样本评测。

当前 repo 不再内置 Unsloth 依赖。训练前需要你自行安装 Unsloth 环境。
