# 训练样本长度计算

本文记录如何使用 `scripts\calc_sample_lengths.py` 统计 SFT 训练样本的 token 长度，用于判断 `training\unsloth_sft.yaml` 中的 `max_seq_length` 应该设置为多少。

## 基本概念

训练样本长度指的是一条样本经过当前模型 tokenizer 处理后的 token 数，不是中文字符数，也不是 JSON 文件中的文本长度。

本工程主 SFT 样本是 tool-aware canonical ReAct 格式。assistant 工具轮包含短 `<think>` 和 JSON tool call，最终答案轮只包含 `<answer>`。脚本会按训练时相同的方式计算长度：

1. 读取 JSONL 中每一行样本。
2. 取出样本里的 `messages` 和 record 级 `tools` 字段。
3. 使用项目 canonical Qwen3 ReAct renderer 渲染成最终训练文本，再用当前模型 tokenizer 统计 token。
4. 对渲染后的文本执行 tokenizer。
5. 统计 `input_ids` 的数量。

因此，统计结果会包含 system、`# Tools`、`<tools>` schema、user、assistant、tool role 渲染出的 `<tool_response>`、Qwen3 对话特殊标记等全部训练上下文，更接近真实训练时的长度。该路径不会使用 tokenizer 原生 `tools=` 模板，也不会给答案轮补空 `<think>`。

## 前置条件

在命令行中进入 `demo` 目录：

```text
cd E:/AI/AgenticRAG-RL/demo
```

当前 uv 环境需要能导入 `transformers` 和 `yaml`：

```text
uv run python -c "import transformers, yaml; print('ok')"
```

默认配置文件为：

```text
E:\AI\AgenticRAG-RL\demo\training\unsloth_sft.yaml
```

默认数据路径来自配置中的：

```yaml
data_path: ./data/novel_eval/sft_zh_unsloth_react_v3/train_cli.jsonl
eval_data_path: ./data/novel_eval/sft_zh_unsloth_react_v3/eval.jsonl
```

默认 tokenizer 来自配置中的：

```yaml
model_name_or_path: unsloth/Qwen3-4B-Instruct-2507
```

## 默认用法

直接运行：

```text
uv run python ./scripts/calc_sample_lengths.py
```

脚本会读取 `training\unsloth_sft.yaml`，自动使用其中的 `model_name_or_path`、`data_path` 和 `max_seq_length`。

## 输出字段含义

`data`：本次统计的数据文件路径。

`model tokenizer`：本次用于分词统计的模型 tokenizer。这个值必须尽量与训练使用的模型一致，否则 token 数可能不准确。

`samples`：有效样本数量。空行会被跳过。

`max_seq_length`：当前配置文件中的训练长度上限。这个字段只用于对照，不会影响统计本身。

`min`：最短样本 token 数。

`p50`：50 分位数，也可以理解为中位数附近的长度。

`avg`：平均 token 数。

`p90`、`p95`、`p99`：分别表示 90%、95%、99% 样本长度不超过该值。它们比平均值更适合判断是否存在长尾样本。

`max`：最长样本 token 数。

`> 2048: 0 (0.0%)`：表示没有样本长度超过 2048 tokens。如果后续重新生成数据后该比例升高，临时降为 `max_seq_length: 2048` 会截断对应样本。

`top 10 longest`：列出最长的若干条样本，`line 10761: 1882` 表示 JSONL 文件第 10761 行样本长度为 1882 tokens。

## 常用参数

### 指定配置文件

```text
uv run python ./scripts/calc_sample_lengths.py --config ./training/unsloth_sft.yaml
```

适合存在多个训练配置时使用。

### 指定数据文件

```text
uv run python ./scripts/calc_sample_lengths.py --data-path ./data/novel_eval/sft_zh_unsloth_react_v3/train_cli.jsonl
```

如果传入相对路径，脚本会按 `demo` 目录解析。

### 指定 tokenizer 模型

```text
uv run python ./scripts/calc_sample_lengths.py --model-name-or-path unsloth/Qwen3-4B-Instruct-2507
```

如果你后续换了基座模型，应使用和训练一致的模型名或本地模型路径。

### 自定义长度阈值

```text
uv run python ./scripts/calc_sample_lengths.py --limits 1024 2048 4096 8192
```

输出会显示超过每个阈值的样本数量和比例。

### 显示更多最长样本

```text
uv run python ./scripts/calc_sample_lengths.py --top 20
```

如果只想看整体统计，不想列出最长样本：

```text
uv run python ./scripts/calc_sample_lengths.py --top 0
```

### 允许远程模型自定义代码

```text
uv run python ./scripts/calc_sample_lengths.py --trust-remote-code
```

只有当模型 tokenizer 明确需要 `trust_remote_code=True` 时再加这个参数。当前 Qwen3 配置通常不需要。

## 如何根据结果调整 max_seq_length

`max_seq_length` 决定单条样本训练时最多保留多少 tokens。超过该长度的部分会被截断。

判断方式：

- 如果 `> 2048` 的比例很高，说明 `max_seq_length: 2048` 会截断较多样本。
- 如果 `max` 小于 2048，设置 `max_seq_length: 2048` 可以完整保留当前数据集。
- 如果重新生成数据后有样本超过 2048，优先截断 tool response 文本；不要先提高 `max_seq_length`。
- 如果训练显存不足，优先降低 `per_device_train_batch_size` 或增加 `gradient_accumulation_steps`，再考虑降低 `max_seq_length`。

以当前 16361 条 v3 CLI 训练样本为例：

```text
> 2048: 0 (0.0%)
> 4096: 0 (0.0%)
```

这说明 `max_seq_length: 2048` 已能覆盖当前数据；主配置已改为 `max_seq_length: 2048`。

## 推荐工作流

1. 生成或更新 SFT 数据。

```text
uv run python ./scripts/convert_sft_to_unsloth.py --input-dir ./data/novel_eval/sft_react_v3 --output-dir ./data/novel_eval/sft_zh_unsloth_react_v3
```

2. 统计样本长度。

```text
uv run python ./scripts/calc_sample_lengths.py --limits 1024 2048 4096
```

3. 根据统计结果修改 `training\unsloth_sft.yaml`。

```yaml
max_seq_length: 2048
```

4. 开始 SFT 训练。

```text
uv run python ./scripts/train_sft_unsloth.py --config ./training/unsloth_sft.yaml
```

## 常见问题

### 为什么 token 数和字符数差很多？

tokenizer 会按模型词表切分文本。中文、英文、数字、标点、特殊标记都会按 tokenizer 规则转成 token，因此 token 数不等于字符数。

### 为什么必须用同一个模型 tokenizer？

不同模型的 tokenizer 规则不同。同一段文本在 Qwen、Llama、Gemma 等模型上的 token 数可能不同。训练长度判断必须使用实际训练模型的 tokenizer。

### 为什么脚本要使用 canonical renderer？

训练时并不是直接拼接原始 `messages` JSON，而是先通过项目 canonical Qwen3 ReAct renderer 渲染成带角色标记、`# Tools`、`<tool_response>` 和特殊标记的文本。只有用同一条 renderer 统计，才接近真实训练长度。

### 超过 max_seq_length 一定会报错吗？

通常不会直接报错，训练框架会截断样本。但被截断的部分不会参与训练。如果 assistant 答案后半段被截断，可能影响模型学习完整输出格式。

### max_seq_length 越大越好吗？

不是。更大的 `max_seq_length` 会增加显存占用和训练时间。应该根据样本长度分布、显存和训练速度选择。当前 v3 CLI 训练集最大约 1882 tokens，因此 2048 可以覆盖，不需要设置到 4096 或 8192。
