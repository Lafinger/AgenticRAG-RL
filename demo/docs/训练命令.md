# Unsloth SFT / RL 训练命令

本文记录在 `demo` 工程中使用 Unsloth 进行 SFT LoRA 和 GRPO/RL 后训练的命令模板。

## 前置约定

- 在 PowerShell 中执行。
- 工作目录必须是 `E:\AI\AgenticRAG-RL\demo`。
- 当前 repo 不再内置 Unsloth 依赖；请先自行在当前 uv 环境中安装 Unsloth、TRL、datasets 等训练栈。
- 当前基座模型默认使用 `Qwen/Qwen3-4B-Instruct-2507`。
- SFT 全量数据位于 `data\novel_eval\sft_zh_unsloth\train.jsonl`。
- Unsloth Studio 建议使用独立切分后的 `data\novel_eval\sft_zh_unsloth\train_studio.jsonl` 和 `data\novel_eval\sft_zh_unsloth\eval.jsonl`。
- SFT 配置位于 `training\unsloth_sft.yaml`。
- GRPO 配置位于 `training\unsloth_grpo.yaml`。

## 进入项目目录

```powershell
cd E:\AI\AgenticRAG-RL\demo
nvidia-smi
```

## 重新生成 Unsloth SFT 数据

```powershell
uv run python .\scripts\convert_sft_to_unsloth.py `
  --input-dir .\data\novel_eval\sft `
  --output-dir .\data\novel_eval\sft_zh_unsloth
```

输出文件：

```text
data\novel_eval\sft_zh_unsloth\train.jsonl
data\novel_eval\sft_zh_unsloth\manifest.json
```

## 切分 Unsloth Studio 训练集和验证集

Unsloth Studio 的训练集和验证集都应使用 `messages` 格式的 SFT 数据。不要直接把 `data\novel_eval\test.jsonl` 填到 Studio 的 Eval dataset，因为它是 QA 评测格式，字段是 `final_question/final_answer/hops`，不是 SFT messages 格式。

下面命令会保留全量 `train.jsonl` 不变，并从中等间隔切出 200 条作为独立验证集。Studio 训练时使用 `train_studio.jsonl`，Eval dataset 使用 `eval.jsonl`；两者样本不重叠。

```powershell
uv run python .\scripts\split_unsloth_studio_data.py `
  --input .\data\novel_eval\sft_zh_unsloth\train.jsonl `
  --train-output .\data\novel_eval\sft_zh_unsloth\train_studio.jsonl `
  --eval-output .\data\novel_eval\sft_zh_unsloth\eval.jsonl `
  --manifest .\data\novel_eval\sft_zh_unsloth\manifest.json `
  --eval-count 200
```

输出文件：

```text
data\novel_eval\sft_zh_unsloth\train_studio.jsonl
data\novel_eval\sft_zh_unsloth\eval.jsonl
```

这份 `eval.jsonl` 与 `train_studio.jsonl` 不重叠，适合在 Studio 中观察 eval loss。正式泛化评测仍应使用 `eval_hf_model.py`、`compare_predictions.py` 和 `run_llm_judge.py` 对 held-out QA 集做训练前后对比。

## SFT LoRA 训练

```powershell
uv run python .\scripts\train_sft_unsloth.py `
  --config .\training\unsloth_sft.yaml `
  --output-dir .\training\outputs\unsloth_sft_qwen3_4b_lora
```

默认配置会同时写出：

```text
training\runs\unsloth_sft_qwen3_4b_lora\
training\outputs\unsloth_sft_qwen3_4b_lora\metrics.jsonl
```

查看 TensorBoard：

```powershell
uv run tensorboard --logdir .\training\runs --port 6006
```

查看项目本地 JSONL dashboard：

```powershell
uv run python .\scripts\watch_unsloth_training.py `
  --output-dir .\training\outputs\unsloth_sft_qwen3_4b_lora `
  --port 8891
```

## 合并 LoRA 到完整模型

```powershell
uv run python .\scripts\export_unsloth_lora.py `
  --config .\training\unsloth_sft.yaml `
  --adapter-path .\training\outputs\unsloth_sft_qwen3_4b_lora `
  --export-dir .\models\Qwen3-4B-Instruct-2507-Unsloth-SFT-merged
```

导出成功后，merged model 位于：

```text
E:\AI\AgenticRAG-RL\demo\models\Qwen3-4B-Instruct-2507-Unsloth-SFT-merged
```

## GRPO / RL 后训练

先启动检索服务：

```powershell
uv run python .\training\tools\retrieval_server.py `
  --corpus .\data\novel\corpus.jsonl `
  --host 127.0.0.1 `
  --port 8790
```

再运行 Unsloth GRPO 入口：

```powershell
uv run python .\scripts\train_grpo_unsloth.py `
  --config .\training\unsloth_grpo.yaml
```

GRPO 默认写出：

```text
training\runs\unsloth_grpo_qwen3_4b\
training\outputs\unsloth_grpo_qwen3_4b\metrics.jsonl
```

也可以用同一个 dashboard 查看：

```powershell
uv run python .\scripts\watch_unsloth_training.py `
  --output-dir .\training\outputs\unsloth_grpo_qwen3_4b `
  --port 8892
```

当前 GRPO 脚本复用 `training\reward_agentic_rag.py` 的 `compute_score` 作为 reward 逻辑。多轮工具调用闭环如果需要完全等价替代旧的远端 rollout，需要后续单独增强。
