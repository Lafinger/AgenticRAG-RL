# Unsloth SFT / RL 训练命令

本文记录在 `demo` 工程中使用 Unsloth 进行 SFT LoRA 和 GRPO/RL 后训练的命令模板。

## 前置约定

- 在 PowerShell 中执行。
- 工作目录必须是 `E:\AI\AgenticRAG-RL\demo`。
- 当前 repo 不再内置 Unsloth 依赖；请先自行在当前 uv 环境中安装 Unsloth、TRL、datasets 等训练栈。
- 当前基座模型默认使用 `Qwen/Qwen3-4B-Instruct-2507`。
- SFT 数据位于 `data\novel_eval\sft_zh_unsloth\train.jsonl`。
- SFT 配置位于 `training\unsloth_sft.yaml`。
- GRPO 配置位于 `training\unsloth_grpo.yaml`。

## 进入项目目录

```powershell
cd E:\AI\AgenticRAG-RL\demo
nvidia-smi
```

## 重新生成 Unsloth SFT 数据

```powershell
uv run python .\scripts\convert_sft_to_unsloth.py `
```

## 生成 Unsloth Studio eval.json

Unsloth Studio 的训练集和验证集都应使用 `messages` 格式的 SFT 数据。不要直接把 `data\novel_eval\test.jsonl` 填到 Studio 的 Eval dataset，因为它是 QA 评测格式，字段是 `final_question/final_answer/hops`，不是 SFT messages 格式。

下面命令会从 `train.jsonl` 中等间隔抽取 200 条样本，生成 Studio 可上传的 JSON 数组文件：

```powershell
@'
import json
from pathlib import Path

input_path = Path("data/novel_eval/sft_zh_unsloth/train.jsonl")
output_path = Path("data/novel_eval/sft_zh_unsloth/eval.json")
eval_count = 200

records = []
with input_path.open("r", encoding="utf-8") as handle:
    all_records = [json.loads(line) for line in handle if line.strip()]

if not all_records:
    raise SystemExit(f"No records found: {input_path}")

step = max(len(all_records) // eval_count, 1)
for index in range(0, len(all_records), step):
    records.append(all_records[index])
    if len(records) >= eval_count:
        break

payload = json.dumps(records, ensure_ascii=False, indent=2)
output_path.write_text(payload.replace("\n", "\r\n") + "\r\n", encoding="utf-8")
print(json.dumps({"input": str(input_path), "output": str(output_path), "source_count": len(all_records), "eval_count": len(records), "step": step}, ensure_ascii=False))
'@ | uv run python -
```

输出文件：

```text
data\novel_eval\sft_zh_unsloth\eval.json
```

这份 `eval.json` 是从当前 SFT 数据抽样生成的，只适合在 Studio 中观察 eval loss 是否稳定下降；正式泛化评测仍应使用 `eval_hf_model.py`、`compare_predictions.py` 和 `run_llm_judge.py` 对 held-out QA 集做训练前后对比。

## SFT LoRA 训练

```powershell
uv run python .\scripts\train_sft_unsloth.py `
  --config .\training\unsloth_sft.yaml `
  --output-dir .\training\outputs\unsloth_sft_qwen3_4b_lora
```

## 合并 LoRA 到完整模型

```powershell
uv run python .\scripts\export_unsloth_lora.py `
  --config .\training\unsloth_sft.yaml `
  --adapter-path .\training\outputs\unsloth_sft_qwen3_4b_lora `
  --export-dir .\models\Qwen3-4B-Instruct-2507-Unsloth-SFT-merged
```

导出成功后，merged model 位于：

```text
E:\AI\AgenticRAG-RL\demo\models\Qwen3-4B-Instruct-2507-Unsloth-SFT-merged
```

## GRPO / RL 后训练

先启动检索服务：

```powershell
uv run python .\training\tools\retrieval_server.py `
  --corpus .\data\novel\corpus.jsonl `
  --host 127.0.0.1 `
  --port 8790
```

再运行 Unsloth GRPO 入口：

```powershell
uv run python .\scripts\train_grpo_unsloth.py `
  --config .\training\unsloth_grpo.yaml
```

当前 GRPO 脚本复用 `training\reward_agentic_rag.py` 的 `compute_score` 作为 reward 逻辑。多轮工具调用闭环如果需要完全等价替代旧的远端 rollout，需要后续单独增强。
