# SFT LoRA 训练命令

本文记录在本机使用外部 LLaMA-Factory 环境执行 `demo` 工程 SFT LoRA 的命令模板。

## 前置约定

- 在 PowerShell 中执行。
- 工作目录必须是 `E:\AI\AgenticRAG-RL\demo`，因为训练配置中的数据路径是相对 `demo` 的。
- 当前命令使用 `Qwen/Qwen3-4B-Instruct-2507` 作为基座模型。
- 数据集配置位于 `data\novel_eval\sft_zh_llamafactory\dataset_info.json`。
- SFT 配置位于 `training\sft_zh_react.yaml`。

## 进入项目目录

```powershell
cd E:\AI\AgenticRAG-RL\demo
llamafactory-cli version
nvidia-smi
```

## 从头训练到新输出目录

推荐使用新 `output_dir`，避免自动从旧 checkpoint 恢复。

```powershell
llamafactory-cli train .\training\sft_zh_react.yaml `
  model_name_or_path=Qwen/Qwen3-4B-Instruct-2507 `
  output_dir=.\training\outputs\sft_qwen3_4b_instruct_2507_lora `
  bf16=true `
  gradient_checkpointing=true
```

正常日志中应出现：

```text
Loading dataset data.json
Num examples = 200
Num Epochs = 3
Total optimization steps = 39
Number of trainable parameters = ...
```

## 覆盖原输出目录重新训练

如果确认不需要保留旧结果，可以覆盖原目录。

```powershell
llamafactory-cli train .\training\sft_zh_react.yaml `
  model_name_or_path=Qwen/Qwen3-4B-Instruct-2507 `
  bf16=true `
  gradient_checkpointing=true `
  overwrite_output_dir=true
```

## 检查 LoRA 产物

```powershell
Get-ChildItem .\training\outputs\sft_qwen3_4b_instruct_2507_lora
```

关键文件：

```text
adapter_model.safetensors
adapter_config.json
trainer_state.json
train_results.json
tokenizer.json
```

## 合并 LoRA 到完整模型

训练完成后，将 LoRA adapter 合并导出为 HuggingFace 完整模型目录。

```powershell
llamafactory-cli export .\training\export_sft.yaml `
  model_name_or_path=Qwen/Qwen3-4B-Instruct-2507 `
  adapter_name_or_path=.\training\outputs\sft_qwen3_4b_instruct_2507_lora `
  export_dir=.\models\Qwen3-4B-Instruct-2507-SFT-merged `
  template=qwen3_nothink `
  finetuning_type=lora
```

导出成功后，merged model 位于：

```text
E:\AI\AgenticRAG-RL\demo\models\Qwen3-4B-Instruct-2507-SFT-merged
```

后续评测或 GRPO 阶段应使用该 merged model 目录作为初始模型。
