# 环境安装

本文记录 `demo` 工程在 Windows 11 + uv 环境下安装 CUDA Torch 与 Unsloth Core 的方法。

## 1. 进入工程目录

```powershell
cd E:\AI\AgenticRAG-RL\demo
```

建议所有命令都在 `demo` 目录执行。

## 2. 创建 / 同步 uv 环境

```powershell
uv sync --inexact
```

说明：

- 当前工程不把 `unsloth` 写入 `pyproject.toml`，因为 Unsloth 环境由你手动安装和维护。
- `--inexact` 表示同步当前工程依赖，但不强制删除环境里额外安装的 Unsloth 训练栈。
- 如果想完全重建环境，可以删除 `.venv` 后重新执行 `uv sync --inexact`。

## 3. 安装 CUDA 版 Torch

本机 RTX 4090 + NVIDIA Driver 560.76 支持 CUDA 12.6，因此安装 cu126 版本：

```powershell
uv pip uninstall torch
uv pip install torch==2.10.0+cu126 --index-url https://download.pytorch.org/whl/cu126
```

选择 `torch==2.10.0+cu126` 的原因：

- `torch 2.10.0+cu126` 能正常使用本机 CUDA。
- 当前 Unsloth 版本要求 `torch>=2.4.0,<2.11.0`。
- `torch 2.11.0+cu126` 虽然 CUDA 可用，但会和当前 Unsloth 依赖约束冲突。

验证：

```powershell
uv run --no-sync python -c "import torch; print(torch.__version__); print(torch.version.cuda); print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0)); x=torch.ones((10,10), device='cuda'); print(x.sum().item(), x.device)"
```

期望输出包含：

```text
2.10.0+cu126
12.6
True
NVIDIA GeForce RTX 4090
100.0 cuda:0
```

如果出现：

```text
Torch not compiled with CUDA enabled
```

说明当前环境装成了 CPU 版 torch，需要重新安装 cu126 版本。

## 4. 安装 Unsloth Core

按 Unsloth Windows 官方安装方式，从 GitHub 安装 Windows 版本：

```powershell
uv pip uninstall unsloth unsloth-zoo
uv pip install "unsloth[windows] @ git+https://github.com/unslothai/unsloth.git"
```

安装完成后检查关键包：

```powershell
uv pip list | Select-String -Pattern "unsloth|torch|triton|xformers|bitsandbytes|transformers|trl|datasets|peft|accelerate"
```

当前验证通过的组合示例：

```text

xformers              0.0.35
bitsandbytes          0.49.2
transformers          5.5.0
trl                   0.24.0
datasets              4.3.0
peft                  0.19.1
unsloth               2026.4.8
unsloth-zoo           2026.4.9
```

## 5. 最小验证

先不要直接跑完整训练样例，先做最小验证：

```powershell
uv run --no-sync python -c "import torch; print(torch.__version__, torch.cuda.is_available()); from unsloth import FastLanguageModel; print('unsloth ok')"
```

期望看到：

```text
2.10.0+cu126 True
unsloth ok
```

也可以验证能加载小模型：

```powershell
@'
import torch
print("torch", torch.__version__, torch.version.cuda, torch.cuda.is_available())
from unsloth import FastLanguageModel
print("unsloth import ok")
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name="unsloth/gemma-3-270m-it",
    max_seq_length=128,
    load_in_4bit=True,
    load_in_8bit=False,
    load_in_16bit=False,
    full_finetuning=False,
)
print("model load ok")
print(type(model).__name__, type(tokenizer).__name__)
'@ | uv run --no-sync python -
```

## 6. 常见提示和问题

### `Redirects are currently not supported in Windows or MacOs`

这是 PyTorch distributed 在 Windows 上的警告，不是致命错误。

示例：

```text
NOTE: Redirects are currently not supported in Windows or MacOs.
```

只要后续脚本继续执行、退出码为 0，就可以忽略。

### `Unsloth: Will patch your computer...`

这是 Unsloth 正常启动提示，不是错误。

### `Flash Attention 2 installation seems to be broken. Using Xformers instead`

Windows 原生环境下常见。Unsloth 会回退到 Xformers，功能可以继续使用，只是性能路径不同。

### 官网完整 `unsloth_check.py` 运行很久

官网示例会下载远程数据集、下载模型并训练 60 step，因此耗时较长。建议先跑本文的最小验证，确认环境可用后，再运行完整训练样例。

## 7. 推荐运行方式

优先使用：

```powershell
uv run --no-sync python .\scripts\train_sft_unsloth.py --help
uv run --no-sync python .\scripts\train_grpo_unsloth.py --help
```

或者显式使用当前 `.venv`：

```powershell
.\.venv\Scripts\python.exe .\scripts\train_sft_unsloth.py --help
```

不建议直接依赖系统 `python`，因为它可能指向 WindowsApps 或其他 Python 环境。

## 8. 后续训练入口

SFT：

```powershell
uv run --no-sync python .\scripts\train_sft_unsloth.py `
  --config .\training\unsloth_sft.yaml
```

导出 merged model：

```powershell
uv run --no-sync python .\scripts\export_unsloth_lora.py `
  --config .\training\unsloth_sft.yaml `
  --adapter-path .\training\outputs\unsloth_sft_qwen3_4b_lora `
  --export-dir .\models\Qwen3-4B-Instruct-2507-Unsloth-SFT-merged
```

GRPO / RL：

```powershell
uv run --no-sync python .\scripts\train_grpo_unsloth.py `
  --config .\training\unsloth_grpo.yaml
```
