# 环境安装

本文记录 `demo` 工程在 Windows 11 + uv 环境下安装 CUDA Torch 与 Unsloth Core 的方法。

当前训练只保留命令行主线 SFT：训练栈需要支持 Qwen3 tokenizer 分词，并使用本项目的 canonical Qwen3 ReAct renderer 与 assistant-only label mask。renderer 负责注入 `# Tools`、包装 `<tool_response>`，label mask 只监督 assistant 输出和对应 turn 结束标记。

## 1. 进入工程目录

```text
cd E:/AI/AgenticRAG-RL/demo
```

建议所有命令都在 `demo` 目录执行。

## 2. 创建 / 同步 uv 环境

```powershell
python `
  ./scripts/setup_env.py
```

说明：

- 当前工程不把 `unsloth` 写入 `pyproject.toml`，因为 Unsloth 环境由你手动安装和维护。
- 初始化脚本会在 `.venv` 不存在时创建环境，并安装 `requirements.txt`。
- 如果后续已经手动安装 Unsloth 训练栈，仍可使用 `uv sync --inexact` 同步项目元信息，避免删除额外包。

## 3. 安装 CUDA 版 Torch

本机 RTX 4090 + NVIDIA Driver 560.76 支持 CUDA 12.6，因此安装 cu126 版本：

```text
uv pip uninstall torch
uv pip install torch==2.10.0+cu126 --index-url https://download.pytorch.org/whl/cu126
```

选择 `torch==2.10.0+cu126` 的原因：

- `torch 2.10.0+cu126` 能正常使用本机 CUDA。
- 当前 Unsloth 版本要求 `torch>=2.4.0,<2.11.0`。
- `torch 2.11.0+cu126` 虽然 CUDA 可用，但会和当前 Unsloth 依赖约束冲突。

验证：

```powershell
$code = @'
import torch
print(torch.__version__)
print(torch.version.cuda)
print(torch.cuda.is_available())
print(torch.cuda.get_device_name(0))
x=torch.ones((10,10), device='cuda')
print(x.sum().item(), x.device)
'@
uv run --no-sync python `
  -c $code
```

期望输出包含：

```text
2.10.0+cu126
12.6
True
NVIDIA GeForce RTX 4090
100.0 cuda:0
```

如果出现：

```text
Torch not compiled with CUDA enabled
```

说明当前环境装成了 CPU 版 torch，需要重新安装 cu126 版本。

## 4. 安装 Unsloth Core

按 Unsloth Windows 官方安装方式，从 GitHub 安装 Windows 版本：

```text
uv pip uninstall unsloth unsloth-zoo
uv pip install "unsloth[windows] @ git+https://github.com/unslothai/unsloth.git"
```

安装完成后检查关键包：

```text
uv pip list
```

重点查看 `unsloth`、`torch`、`triton`、`xformers`、`bitsandbytes`、`transformers`、`trl`、`datasets`、`peft`、`accelerate` 等包的版本。

当前验证通过的组合示例：

```text

xformers              0.0.35
bitsandbytes          0.49.2
transformers          5.5.0
trl                   0.24.0
datasets              4.3.0
peft                  0.19.1
unsloth               2026.4.8

```

## 4.5. 协议模板验证

主 SFT 链路依赖项目 canonical Qwen3 ReAct renderer，而不是直接依赖 tokenizer 原生 `tools=` 模板。训练栈必须支持：

- tokenizer 对 canonical renderer 输出文本做稳定分词。
- renderer 自动注入 `# Tools` 和 `<tools>` schema。
- `tool` role 渲染为用户侧 `<tool_response>...</tool_response>`。
- 工具轮必须是短 `<think>` + JSON `<tool_call>`；最终答案轮保持 `<answer>...</answer>`，不会自动补空 `<think>`。
- 项目脚本里的 assistant-only label mask 只监督 assistant token。

安装或升级 `transformers`、`trl`、`unsloth` 后，建议先运行目标单测确认模板和 mask 没有退化：

```powershell
uv run pytest `
  ./tests/test_sft_label_mask.py `
  ./tests/test_eval_hf_agentic.py
```

## 6. 常见提示和问题

### `Redirects are currently not supported in Windows or MacOs`

这是 PyTorch distributed 在 Windows 上的警告，不是致命错误。

示例：

```text
NOTE: Redirects are currently not supported in Windows or MacOs.
```

只要后续脚本继续执行、退出码为 0，就可以忽略。

### `Unsloth: Will patch your computer...`

这是 Unsloth 正常启动提示，不是错误。

### `Flash Attention 2 installation seems to be broken. Using Xformers instead`

Windows 原生环境下常见。Unsloth 会回退到 Xformers，功能可以继续使用，只是性能路径不同。

### 官网完整 `unsloth_check.py` 运行很久

官网示例会下载远程数据集、下载模型并训练 60 step，因此耗时较长。建议先跑本文的最小验证，确认环境可用后，再运行完整训练样例。

## 7. 推荐运行方式

优先使用：

```powershell
uv run --no-sync python `
  ./scripts/train_sft_unsloth.py `
  --help
uv run --no-sync python `
  ./scripts/train_grpo_unsloth.py `
  --help
```

或者显式使用当前 `.venv`：

```powershell
./.venv/Scripts/python.exe `
  ./scripts/train_sft_unsloth.py `
  --help
```

不建议直接依赖系统 `python`，因为它可能指向 WindowsApps 或其他 Python 环境。

## 8. 后续训练入口

SFT：

```powershell
uv run --no-sync python `
  ./scripts/train_sft_unsloth.py `
  --config ./training/unsloth_sft.yaml
```

`training\unsloth_sft.yaml` 默认使用 `train_cli.jsonl` 训练集和 `eval.jsonl` 验证集；标准命令会按配置中的 `eval_steps` 记录 eval loss。

如果启用 SwanLab，断点续训依赖 `swanlab>=0.7.12` 的 `id/resume` 参数。默认配置会写入并复用：

```text
training\outputs\unsloth_sft_qwen3_4b_lora_react_v3\swanlab_run_id.txt
```

续训时脚本会设置 `SWANLAB_RUN_ID` 和 `SWANLAB_RESUME=allow`，并把本地历史指标回放到同一个 SwanLab run。若要确认 SwanLab SDK 支持这些参数：

```powershell
$code = @'
import swanlab, inspect
print(swanlab.__version__)
print(inspect.signature(swanlab.init))
'@
uv run python `
  -c $code
```

导出 merged model：

```powershell
uv run --no-sync python `
  ./scripts/export_unsloth_lora.py `
  --config ./training/unsloth_sft.yaml `
  --adapter-path ./training/outputs/unsloth_sft_qwen3_4b_lora_react_v3 `
  --export-dir ./models/Qwen3-4B-Instruct-2507-Unsloth-SFT-react-v3-merged
```

GRPO / RL：

```powershell
uv run --no-sync python `
  ./scripts/train_grpo_unsloth.py `
  --config ./training/unsloth_grpo.yaml
```
