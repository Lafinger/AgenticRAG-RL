# SwanLab + 项目 JSONL Dashboard 训练观测

本文记录本工程在 Unsloth 自定义 SFT / GRPO 训练中的观测方案。当前默认把标准训练指标上报到 SwanLab 云端，同时保留项目自己的 JSONL dashboard 用于本地样本追溯和异常 step 分析。

## 结论

推荐采用两层观测：

| 层级 | 目标 | 工具 |
| --- | --- | --- |
| 标准指标层 | 实时查看 loss、eval loss、learning rate、grad norm、训练配置和实验对比 | SwanLab 云端项目 `agentic-rag-rl` |
| 样本追溯层 | 解释异常 step，反查 `source_line`、`sample_id`、question、answer、gold chunks、token 长度 | 项目本地 JSONL dashboard |

SwanLab 只记录标量指标和训练配置，不上传 `step_sample_trace.jsonl` 中的问题、答案、`source_line`、`gold_chunks` 等样本文本明细。这些明细只保存在本地训练输出目录。

## 前置准备

安装依赖：

```powershell
Set-Location E:\AI\AgenticRAG-RL\demo
uv pip install -r .\requirements.txt
```

首次使用 SwanLab 云端模式前登录：

```powershell
uv run swanlab login
```

如果当前环境手动安装了 Unsloth 训练栈，并且不希望 `uv run` 自动同步依赖，训练命令仍建议使用 `uv run --no-sync python ...`。只要同一个环境中已安装 `swanlab>=0.7.12`，Trainer 集成即可上报。

## 默认配置

`training\unsloth_sft.yaml` 和 `training\unsloth_grpo.yaml` 默认包含：

```yaml
report_to:
  - swanlab
swanlab_project: agentic-rag-rl
swanlab_mode: cloud
swanlab_logdir: ./training/swanlab
metrics_output: ./training/outputs/.../metrics.jsonl
```

不再配置本地事件目录，训练脚本不会默认生成额外的 events 文件。

默认值保留云端模式。需要只使用本地 SwanLab 时，不要移除 `report_to: [swanlab]`，只把 `swanlab_mode` 覆盖为 `local`；这样 SwanLab 会写本地日志，不上传云端。

## 普通 SFT / GRPO 训练观测

SFT：

```powershell
uv run --no-sync python .\scripts\train_sft_unsloth.py `
  --config .\training\unsloth_sft.yaml `
  --output-dir .\training\outputs\unsloth_sft_qwen3_4b_lora
```

GRPO：

```powershell
uv run --no-sync python .\scripts\train_grpo_unsloth.py `
  --config .\training\unsloth_grpo.yaml
```

这两个 Trainer 路径同时做两件事：

- 通过 `report_to: [swanlab]` 使用 Transformers / TRL 原生 SwanLab 集成，上报标准标量指标。
- 通过 `training.monitoring.JsonlMetricsCallback` 写 `metrics.jsonl`，供本地 dashboard 读取。

可用命令行覆盖 SwanLab 参数：

```powershell
uv run --no-sync python .\scripts\train_sft_unsloth.py `
  --config .\training\unsloth_sft.yaml `
  --swanlab-project agentic-rag-rl `
  --swanlab-mode cloud `
  --swanlab-logdir .\training\swanlab `
  --swanlab-experiment-name sft-qwen3-4b-lora
```

本地 SwanLab 模式：

```powershell
uv run --no-sync python .\scripts\train_sft_unsloth.py `
  --config .\training\unsloth_sft.yaml `
  --swanlab-mode local `
  --swanlab-logdir .\training\swanlab `
  --swanlab-experiment-name sft-qwen3-4b-lora-local

uv run swanlab watch -l .\training\swanlab
```

未指定 `--swanlab-workspace` 时使用当前 SwanLab 账号默认空间。

## 可追溯 SFT 训练观测

可追溯训练入口使用自定义训练循环，不能依赖 Trainer 自动回调，因此通过 `SwanLabScalarLogger` 手动记录标量：

```powershell
uv run --no-sync python .\scripts\train_sft_unsloth_trace.py `
  --config .\training\unsloth_sft.yaml `
  --data-path .\data\novel_eval\sft_zh_unsloth\train_studio.jsonl `
  --eval-data-path .\data\novel_eval\sft_zh_unsloth\eval.jsonl `
  --output-dir .\training\outputs\unsloth_sft_qwen3_4b_lora_trace `
  --per-device-train-batch-size 2 `
  --gradient-accumulation-steps 4 `
  --learning-rate 1.0e-4 `
  --eval-steps 105 `
  --max-grad-norm 1.0
```

上传到 SwanLab 的字段只包括：

| 本地字段 | SwanLab 指标 |
| --- | --- |
| `loss` | `train/loss` |
| `grad_norm` | `train/grad_norm` |
| `learning_rate` | `train/learning_rate` |
| `eval_loss` | `eval/loss` |

本地仍写出：

```text
training\outputs\unsloth_sft_qwen3_4b_lora_trace\trace_metrics.jsonl
training\outputs\unsloth_sft_qwen3_4b_lora_trace\step_sample_trace.jsonl
training\outputs\unsloth_sft_qwen3_4b_lora_trace\trace_training_state.json
training\outputs\unsloth_sft_qwen3_4b_lora_trace\trace_run_config.json
```

`trace_run_config.json` 记录 `swanlab_logdir` 等实验配置，不记录样本文本。

## 本地 JSONL Dashboard

标准曲线看 SwanLab；异常样本追溯看本地 dashboard：

```powershell
uv run python .\scripts\watch_unsloth_training.py `
  --output-dir .\training\outputs\unsloth_sft_qwen3_4b_lora_trace `
  --host 127.0.0.1 `
  --port 8891 `
  --refresh-seconds 3
```

浏览器打开：

```text
http://127.0.0.1:8891
```

观察器按优先级读取：

1. `trace_metrics.jsonl`
2. `trace_training_state.json`
3. `step_sample_trace.jsonl`
4. `metrics.jsonl`
5. `checkpoint-*\trainer_state.json`
6. `trainer_state.json`

普通 SFT / GRPO 没有 trace 文件时，dashboard 会使用 `metrics.jsonl` 展示标准曲线。可追溯 SFT 有 trace 文件时，可以点击异常 step 查看本地样本明细。

## 数据边界

允许上传 SwanLab：

- `loss`
- `eval_loss`
- `grad_norm`
- `learning_rate`
- Trainer / run config 中的非样本文本配置

只保留本地：

- `sample_ids`
- `source_lines`
- `question`
- `answer`
- `hop_count`
- `gold_chunks`
- `step_sample_trace.jsonl` 中的完整 micro-batch 样本明细

## 故障排查

| 现象 | 处理 |
| --- | --- |
| 提示未安装 SwanLab | 执行 `uv pip install "swanlab>=0.7.12"` 或 `uv pip install -r .\requirements.txt` |
| SwanLab 没有实验 | 先执行 `uv run swanlab login`，再确认 `report_to` 包含 `swanlab` |
| 本地 dashboard 没有曲线 | 检查训练输出目录下是否存在 `metrics.jsonl` 或 `trace_metrics.jsonl` |
| dashboard 看不到样本明细 | 只有 `train_sft_unsloth_trace.py` 会写 `step_sample_trace.jsonl` |

## 外部接口依据

- SwanLab Unsloth 集成文档说明可通过 `SwanLabCallback` 或 TRL / Transformers 集成记录 Unsloth 训练。
- SwanLab Transformers / TRL 文档说明 `transformers>=4.50.0` 可直接使用 `report_to="swanlab"`。
- Transformers `TrainerCallback` 提供 `on_log` 等回调点，适合把 Trainer logs 追加到项目自己的 JSONL。
