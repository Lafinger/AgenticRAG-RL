# 训练与观测

本文只记录 demo 当前保留的主线训练流程：canonical ReAct SFT -> Unsloth LoRA -> merged model -> Agent loop 测评。所有训练入口都通过命令行执行。

训练、长度统计和 Agent loop 测评都使用项目 canonical Qwen3 ReAct renderer。该 renderer 统一注入 `# Tools`、包装 `<tool_response>`，并强制主线工具轮是短 `<think>` + JSON `<tool_call>`；不再直接使用 tokenizer 原生 `tools=` 模板。

## 主线数据准备

主线 ReAct SFT 的 assistant 工具轮包含从 Oracle plan 生成的短 `<think>` 和标准 JSON tool call。第 1 跳使用“要回答最终问题，先查”，后续跳使用“已获得上一跳线索”：

```text
<think>要回答最终问题，先查：卖饼老者自称是哪个集市的人？</think>
<tool_call>
{"name":"keyword_search","arguments":{"query":"卖饼老者自称是哪个集市的人？"}}
</tool_call>
```

最终答案仍使用：

```text
<answer>最终答案</answer>
```

重新生成数据时按下面顺序执行：

```text
uv run python ./scripts/build_oracle_traces.py --qa ./data/novel_eval/qa_pairs.jsonl --corpus ./data/novel/corpus.jsonl --output ./data/novel_eval/traces_oracle_zh.jsonl --use-zh

uv run python ./scripts/trace_to_sft.py --input ./data/novel_eval/traces_oracle_zh.jsonl --output-dir ./data/novel_eval/sft --lang zh

uv run python ./scripts/convert_sft_to_unsloth.py --input-dir ./data/novel_eval/sft --output-dir ./data/novel_eval/sft_zh_unsloth

uv run python ./scripts/split_sft_train_eval.py --input ./data/novel_eval/sft_zh_unsloth/train.jsonl --train-output ./data/novel_eval/sft_zh_unsloth/train_cli.jsonl --eval-output ./data/novel_eval/sft_zh_unsloth/eval.jsonl --manifest ./data/novel_eval/sft_zh_unsloth/manifest.json --eval-count 200
```

当前主线数据规模：

| 文件 | 数量 | 用途 |
| --- | ---: | --- |
| `data\novel_eval\sft_zh_unsloth\train.jsonl` | 6000 | 全量 SFT 导出，包含 `full_trace + finalization_only` |
| `data\novel_eval\sft_zh_unsloth\train_cli.jsonl` | 5800 | 命令行训练输入 |
| `data\novel_eval\sft_zh_unsloth\eval.jsonl` | 200 | 训练过程 eval loss 验证集 |

`train_cli.jsonl` 和 `eval.jsonl` 由同一个全量导出切分得到，二者不重叠。`data\novel_eval\test.jsonl` 是最终 held-out Agent loop 测评集，不进入 SFT 训练。

## 长度统计

训练前必须用同一 tokenizer 和同一 canonical Qwen3 ReAct renderer 统计长度：

```text
uv run python ./scripts/calc_sample_lengths.py --config ./training/unsloth_sft.yaml --limits 1024 2048 4096
```

最近一次主线数据统计结果：

```text
samples: 5800
max_seq_length: 2048
min: 810 p50: 1551 avg: 1574.9 p90: 1927 p95: 1954 p99: 1996 max: 2045
> 1024: 5756 (99.2%)
> 2048: 0 (0.0%)
> 4096: 0 (0.0%)
```

因此当前 `training\unsloth_sft.yaml` 保持 `max_seq_length: 2048`。

## 协议诊断

训练前建议先检查 canonical renderer 和 assistant-only labels 是否符合主线协议：

```text
uv run python ./scripts/diagnose_sft_protocol.py --config ./training/unsloth_sft.yaml --max-samples 200
```

重点看 `tool_turn_think_rate=1.0`、`rendered_empty_answer_think=0`、`tool_response_label_leaks=0`。如果任一项异常，先修数据或 renderer，不要直接训练。

## 标准 LoRA 训练

主线配置文件固定为：

```text
training\unsloth_sft.yaml
```

核心训练参数：

| 参数 | 当前值 |
| --- | --- |
| base model | `unsloth/Qwen3-4B-Instruct-2507` |
| data path | `data/novel_eval/sft_zh_unsloth/train_cli.jsonl` |
| eval path | `data/novel_eval/sft_zh_unsloth/eval.jsonl` |
| max seq length | `2048` |
| LoRA rank | `64` |
| learning rate | `1e-4` |
| epoch | `3` |
| batch / grad accum | `2 x 8` |
| packing | `false` |

从头训练：

```text
uv run --no-sync python ./scripts/train_sft_unsloth.py --config ./training/unsloth_sft.yaml
```

继续训练或从最新 checkpoint 恢复时，默认使用 `--resume`；输出目录仍保持在配置中的 `training\outputs\unsloth_sft_qwen3_4b_lora`：

```text
uv run --no-sync python ./scripts/train_sft_unsloth.py --config ./training/unsloth_sft.yaml --resume
```

续训时脚本会为当前 `output_dir` 复用或创建 `swanlab_run_id.txt`，设置 `SWANLAB_RUN_ID` 和 `SWANLAB_RESUME=allow`，并先把本地 `metrics.jsonl` 或 checkpoint `trainer_state.json` 中 `step <= checkpoint_step` 的历史标量回放到同一个 SwanLab run，然后再继续上传新日志。回放状态写入：

```text
training\outputs\unsloth_sft_qwen3_4b_lora\swanlab_history_replay_state.json
```

如果当前已经有旧续训进程在运行，代码修改不能热修复那个进程的 SwanLab 初始化。需要停止后从最新 checkpoint 重新启动一次，新的脚本会自动回放历史曲线并继续训练。

手动补传历史指标但不重新训练：

```text
uv run python ./scripts/replay_swanlab_history.py --output-dir ./training/outputs/unsloth_sft_qwen3_4b_lora --checkpoint ./training/outputs/unsloth_sft_qwen3_4b_lora/checkpoint-495 --dry-run
```

确认 `count/first_step/last_step` 正常后去掉 `--dry-run`。如果要接到已有 SwanLab run，额外传：

```text
--swanlab-run-id <已有 run id>
```

## 可追溯训练

需要定位 loss 或 grad norm 异常样本时，使用 trace 训练入口：

```text
uv run --no-sync python ./scripts/train_sft_unsloth_trace.py --config ./training/unsloth_sft.yaml
```

追溯输出：

```text
training\outputs\unsloth_sft_qwen3_4b_lora_trace\step_sample_trace.jsonl
training\outputs\unsloth_sft_qwen3_4b_lora_trace\sample_loss_trace.jsonl
training\outputs\unsloth_sft_qwen3_4b_lora_trace\trace_summary.json
```

按 step 查看样本：

```text
uv run python ./scripts/inspect_sft_trace.py --trace ./training/outputs/unsloth_sft_qwen3_4b_lora_trace --step 735
```

按最高 grad norm 查看样本：

```text
uv run python ./scripts/inspect_sft_trace.py --trace ./training/outputs/unsloth_sft_qwen3_4b_lora_trace --top-grad 20
```

## 训练观测

训练脚本会记录：

| 指标 | 用途 |
| --- | --- |
| `train/loss` | 判断 SFT 是否收敛 |
| `eval/loss` | 判断 held-out train/eval split 上是否过拟合 |
| `train/grad_norm` | 观察梯度突刺 |
| `train/learning_rate` | 检查调度是否按预期衰减 |
| `train/epoch` | 确认训练进度 |
| `train/global_step` | 对齐 checkpoint 和 trace |

本地轮询输出目录：

```text
uv run python ./scripts/watch_unsloth_training.py --output-dir ./training/outputs/unsloth_sft_qwen3_4b_lora --interval 30
```

SwanLab 页面重点看两点：

- loss 应先快速下降，再进入缓慢收敛区间。
- grad norm 不应长期持续升高；偶发尖峰需要结合 trace 样本确认是否由长样本或异常标签导致。
- 断点续训后的 SwanLab 曲线应从 step 5 等早期历史点开始连续显示，而不是只显示续训后的新 step。

## 导出 merged model

LoRA 训练完成后，导出主线 merged model：

```text
uv run --no-sync python ./scripts/export_unsloth_lora.py --config ./training/unsloth_sft.yaml --adapter-path ./training/outputs/unsloth_sft_qwen3_4b_lora --export-dir ./models/Qwen3-4B-Instruct-2507-Unsloth-SFT-merged
```

也可以导出某个 checkpoint：

```text
uv run --no-sync python ./scripts/export_unsloth_lora.py --config ./training/unsloth_sft.yaml --adapter-path ./training/outputs/unsloth_sft_qwen3_4b_lora/checkpoint-735 --export-dir ./models/Qwen3-4B-Instruct-2507-Unsloth-SFT-ckpt735-merged
```

## 训练后检查

训练完成后按下面顺序判断是否进入下一步：

1. `eval/loss` 没有明显发散。
2. 指定 checkpoint 能被 `eval_hf_agentic.py --adapter` 正常加载。
3. 5 条 smoke 样本中工具轮为短 `<think>` 加 JSON `<tool_call>`，而不是裸查询文本。
4. 50 条 Agent loop 中 `valid_tool_call_rate` 保持可用，`malformed_tool_fragment_rate` 下降，`answer_tag_rate` 上升。

Agent loop 命令和 checkpoint 选择方法见 `docs\训练测评.md`。

## 常见问题

| 现象 | 处理 |
| --- | --- |
| `max_seq_length` 截断样本 | 先重新跑长度统计；当前主线数据不应超过 2048 |
| checkpoint 路径被当成 Hugging Face repo id | 使用当前 `eval_hf_agentic.py`，它会自动把存在的本地 `--model` / `--adapter` 路径解析为绝对路径 |
| 模型输出 `<tool_call>查询文本</tool_call>` | 这是格式错误，应通过主线 ReAct SFT 数据和重新训练修复 |
| 模型只调用工具不回答 | 检查 finalization_only 样本是否存在，并确认 assistant turn 结束 token 被纳入 loss |
| Agent loop 达到最大轮次 | 先查看 `raw_turns`，确认是不会停、格式错，还是检索证据不足 |
