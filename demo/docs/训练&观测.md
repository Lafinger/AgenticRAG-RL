# 训练 & 观测

本文作为 Unsloth SFT / GRPO 训练和观测的统一入口。训练后的模型测评见 `docs\训练测评.md`，按 step 追溯异常样本的细节见 `docs\训练追溯.md`。

## 前置约定

- 在 PowerShell 中执行命令。
- 工作目录固定为 `E:\AI\AgenticRAG-RL\demo`。
- 当前基座模型默认是 `unsloth/Qwen3-4B-Instruct-2507`。
- 当前 repo 不内置完整 Unsloth 训练栈；训练前先按 `docs\环境安装.md` 安装 CUDA Torch、Unsloth、TRL、datasets、PEFT 等依赖。
- SFT 配置统一从 `training\unsloth_sft.yaml` 读取，GRPO 配置统一从 `training\unsloth_grpo.yaml` 读取。
- 如果当前 `.venv` 已手动安装 Unsloth 训练栈，训练命令优先使用 `uv run --no-sync python ...`，避免 `uv run` 自动同步时影响手动安装的包。

进入项目目录并检查训练栈：

```powershell

uv run --no-sync python -c "import torch; print(torch.__version__, torch.cuda.is_available()); from unsloth import FastLanguageModel; print('unsloth ok')"
```

首次使用 SwanLab 云端模式前登录：

```powershell
uv run swanlab login
```

## SFT 数据准备

如果只是因为 assistant-only loss mask 变更而重新训练，不需要重新生成 SFT 数据；新的 label mask 在训练时动态生成。只有 Oracle traces、SFT 中间目录或 corpus/chunk_id 发生变化时，才重新导出 Unsloth SFT 数据。

重新导出 Unsloth messages 数据：

```powershell
uv run python .\scripts\convert_sft_to_unsloth.py `
  --input-dir .\data\novel_eval\sft `
  --output-dir .\data\novel_eval\sft_zh_unsloth
```

输出：

```text
data\novel_eval\sft_zh_unsloth\train.jsonl
data\novel_eval\sft_zh_unsloth\manifest.json
```

Unsloth Studio 建议使用从全量 SFT 数据切出的独立 train/eval 文件，不要把 `data\novel_eval\test.jsonl` 作为 Studio eval dataset；`test.jsonl` 是最终 held-out QA 测评集，不是 SFT messages 格式。

```powershell
uv run python .\scripts\split_unsloth_studio_data.py `
  --input .\data\novel_eval\sft_zh_unsloth\train.jsonl `
  --train-output .\data\novel_eval\sft_zh_unsloth\train_studio.jsonl `
  --eval-output .\data\novel_eval\sft_zh_unsloth\eval.jsonl `
  --manifest .\data\novel_eval\sft_zh_unsloth\manifest.json `
  --eval-count 200
```

训练前可检查样本长度：

```powershell
uv run python .\scripts\calc_sample_lengths.py `
  --config .\training\unsloth_sft.yaml `
  --limits 1024 2048 4096
```

## SFT 训练

当前标准 SFT 和可追溯 SFT 都默认启用 assistant-only loss mask：训练 loss 和 eval loss 只统计 assistant token，system/user/tool_response 和 padding 的 label 都是 `-100`。这避免模型学习复读用户问题或检索返回的 `<tool_response>`。

SFT 关键超参统一由 `training\unsloth_sft.yaml` 管理：

```text
model_name_or_path: unsloth/Qwen3-4B-Instruct-2507
per_device_train_batch_size: 2
gradient_accumulation_steps: 8
learning_rate: 1.0e-4
eval_steps: 105
max_grad_norm: 1.0
```

标准 SFT LoRA 训练：

```powershell
uv run --no-sync python .\scripts\train_sft_unsloth.py `
  --config .\training\unsloth_sft.yaml `
  --output-dir .\training\outputs\unsloth_sft_qwen3_4b_lora
```

如果需要在训练时精确记录每个 optimizer step 对应哪些样本，使用可追溯 SFT 入口：

```powershell
uv run --no-sync python .\scripts\train_sft_unsloth_trace.py `
  --config .\training\unsloth_sft.yaml `
  --data-path .\data\novel_eval\sft_zh_unsloth\train_studio.jsonl `
  --eval-data-path .\data\novel_eval\sft_zh_unsloth\eval.jsonl `
  --output-dir .\training\outputs\unsloth_sft_qwen3_4b_lora_trace
```

可追溯入口仍从 YAML 读取 batch、梯度累积、学习率、eval 间隔和 grad norm 裁剪；命令行参数只用于临时覆盖。详细 trace 字段和异常 step 排查见 `docs\训练追溯.md`。

## 训练观测

默认配置会同时写 SwanLab 标量指标和项目本地 JSONL 指标：

```yaml
report_to:
  - swanlab
swanlab_project: agentic-rag-rl
swanlab_mode: cloud
swanlab_logdir: ./training/swanlab
metrics_output: ./training/outputs/.../metrics.jsonl
```

默认是 SwanLab 云端模式。如果只使用本地 SwanLab，不要移除 `report_to: [swanlab]`，只在命令行覆盖模式和日志目录：

```powershell
uv run --no-sync python .\scripts\train_sft_unsloth.py `
  --config .\training\unsloth_sft.yaml `
  --swanlab-mode local `
  --swanlab-logdir .\training\swanlab `
  --swanlab-experiment-name sft-qwen3-4b-local

uv run swanlab watch -l .\training\swanlab
```

支持的 `swanlab_mode` 只有：

```text
cloud
local
offline
disabled
```

本地 JSONL dashboard 用于看曲线和追溯异常 step：

```powershell
uv run python .\scripts\watch_unsloth_training.py `
  --output-dir .\training\outputs\unsloth_sft_qwen3_4b_lora_trace `
  --host 127.0.0.1 `
  --port 8891 `
  --refresh-seconds 3
```

浏览器打开：

```text
http://127.0.0.1:8891
```

dashboard 按优先级读取：

1. `trace_metrics.jsonl`
2. `trace_training_state.json`
3. `step_sample_trace.jsonl`
4. `metrics.jsonl`
5. `checkpoint-*\trainer_state.json`
6. `trainer_state.json`

上传到 SwanLab 的内容只包括标量指标和非样本文本配置：

```text
train/loss
train/grad_norm
train/learning_rate
eval/loss
```

只保存在本地的内容包括：

```text
sample_ids
source_lines
question
answer
hop_count
gold_chunks
step_sample_trace.jsonl 中的完整 micro-batch 样本明细
```

## LoRA 合并与 Studio 接入

训练完成后导出 merged model：

```powershell
uv run --no-sync python .\scripts\export_unsloth_lora.py `
  --config .\training\unsloth_sft.yaml `
  --adapter-path .\training\outputs\unsloth_sft_qwen3_4b_lora `
  --export-dir .\models\Qwen3-4B-Instruct-2507-Unsloth-SFT-merged
```

如果使用可追溯训练输出作为 adapter，把 `--adapter-path` 改为：

```text
.\training\outputs\unsloth_sft_qwen3_4b_lora_trace
```

建议在 Unsloth Studio 的 Custom Folder / Scan Folder 中添加：

```text
E:\AI\AgenticRAG-RL\demo\models
```

如果必须复制到 Studio 自己的 models 目录，使用辅助脚本：

```powershell
uv run python .\scripts\copy_merged_model_to_studio.py `
  --source "E:\AI\AgenticRAG-RL\demo\models\Qwen3-4B-Instruct-2507-Unsloth-SFT-merged"
```

## GRPO / RL 训练

GRPO 前先启动检索服务：

```powershell
uv run python .\training\tools\retrieval_server.py `
  --corpus .\data\novel\corpus.jsonl `
  --host 127.0.0.1 `
  --port 8790
```

再运行 Unsloth GRPO 入口：

```powershell
uv run --no-sync python .\scripts\train_grpo_unsloth.py `
  --config .\training\unsloth_grpo.yaml
```

GRPO 默认写出：

```text
SwanLab 项目：agentic-rag-rl
training\outputs\unsloth_grpo_qwen3_4b\metrics.jsonl
```

查看 GRPO dashboard：

```powershell
uv run python .\scripts\watch_unsloth_training.py `
  --output-dir .\training\outputs\unsloth_grpo_qwen3_4b `
  --port 8892
```

当前 GRPO 脚本复用 `training\reward_agentic_rag.py` 的 `compute_score` 作为 reward 逻辑。多轮工具调用闭环如果需要完全等价替代旧的远端 rollout，需要后续单独增强。

## 故障排查

| 现象 | 处理 |
| --- | --- |
| 提示未安装 Unsloth / TRL / datasets | 按 `docs\环境安装.md` 安装训练栈，训练命令使用 `uv run --no-sync python ...` |
| 提示未安装 SwanLab | 执行 `uv pip install "swanlab>=0.7.12"` 或 `uv pip install -r .\requirements.txt` |
| SwanLab 云端没有实验 | 先执行 `uv run swanlab login`，再确认 `report_to` 包含 `swanlab` |
| 只想本地记录 SwanLab | 使用 `--swanlab-mode local --swanlab-logdir .\training\swanlab` |
| `swanlab_mode` 报错 | 只使用 `cloud/local/offline/disabled` |
| dashboard 没有曲线 | 检查输出目录下是否存在 `metrics.jsonl` 或 `trace_metrics.jsonl` |
| dashboard 看不到样本明细 | 只有 `train_sft_unsloth_trace.py` 会写 `step_sample_trace.jsonl` |
| 训练后模型复读问题或 tool_response | 确认重新训练使用了当前 assistant-only loss mask，不要继续使用旧 adapter/merged model |

## 下一步测评

SFT 训练后不要只看训练 loss。推荐继续执行：

- 直接答案评测：`docs\训练测评.md` 中的 direct-answer 流程。

- 异常样本追溯：`docs\训练追溯.md`，按 grad norm 或 step 反查训练样本。
