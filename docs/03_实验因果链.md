# 实验因果链

这份项目文档最值得学的不是“我做了哪些模块”，而是“我如何定位系统为什么没提升”。

## Agent 系统因果链

### 起点

- Baseline PEV 已经能工作
- 但 `iter=2/3` 的样本并没有稳定优于 `iter=1`

### 观察

- 限制最大迭代次数并不能解决问题
- 激进的 evidence dedup 反而伤害效果
- 真正有效的是：更好的 `replan feedback` 和多工具协同

### 结论

- 问题不是“Verifier 太严格”
- 问题是“Verifier 判不够之后，Planner 仍然搜不到新东西”

### 对你学习的意义

以后你看 Agent 系统，不要只盯节点名字，要盯：

- 失败后有没有获得新信息
- 新一轮计划有没有真的改搜索策略
- 多轮循环是不是只是重复召回

## 检索系统因果链

### 观察

- `keyword_search` 的消融损伤最大
- `semantic_search` 也有贡献
- `semantic_only` 下滑明显

### 结论

金融语料是典型的“精确匹配不能丢”的领域，原因是大量问题本质上依赖：

- 人名
- 公司名
- 数字
- 表格字段

### 对你学习的意义

不要默认 dense 检索一定比 BM25 更高级。领域不同，主力工具也不同。

## RL 因果链

### 现象

- 直接优化 correctness，模型会走捷径
- faithfulness 上去时，correctness 又容易掉

### 原因

因为“基于证据回答”与“最后答对”不是平级能力，而是前者支撑后者。

### 最有效解法

两阶段课程学习：

1. 先学 grounding / faithfulness
2. 再学 correctness

### 对你学习的意义

以后碰到 RL 任务，不要一上来就追主指标。先问：

- 模型有没有基础行为前提
- reward 是否在鼓励错误捷径
- 当前 reward 的非零信号密度够不够

## 你应该复述出来的三句话

1. `PEV` 提升的关键不是多迭代，而是更好的 `replan feedback`。
2. 金融语料里的检索主力通常不是单一 dense，而是 `BM25 + dense + rerank`。
3. RL 必须先学 grounding，再学 correctness，否则模型容易 reward hack。
