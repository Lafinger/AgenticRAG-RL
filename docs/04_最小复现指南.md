# 最小复现指南

这一步的目标不是“完整复刻论文工程”，而是先把主链跑通。

## 当前最小复现范围

已经落地到本目录代码里的范围：

- PDF 文本抽取
- 简单 chunk 切分
- `keyword_search`
- `dense_search`
- `RRF` 融合
- 轻量 rerank
- 单轮问答与多跳示例

暂时不做：

- 知识图谱三元组抽取
- 外部 LLM Judge
- 大规模合成 pipeline
- 完整训练

## 先跑自带样例

```powershell
python scripts/run_retrieval_demo.py
```

这个脚本会读取 `data/demo_financial/corpus.jsonl`，对几条问题分别执行：

- `keyword_search`
- `dense_search`
- `hybrid_search`

## 再把自己的 PDF 变成 chunk

```powershell
python scripts/parse_pdf_to_chunks.py `
  --pdf ".\你的文档.pdf" `
  --output ".\data\custom\chunks.jsonl"
```

默认切块逻辑是“按段落聚合 + 固定字符窗”，适合先理解流程，不适合直接上生产。

## 为什么示例里没有强绑 BGE-M3 + FAISS

因为你当前目标是 `单卡/CPU 友好`。所以这里的设计是：

- 先让流程跑通
- 再按需切到更贴近原项目的 embedding / reranker

代码里已经预留了替换位：

- dense 检索可切到 `sentence-transformers` 或你自己的 embedding 服务
- reranker 可切到 `CrossEncoder`

## 你做完这一步后应该会什么

- 知道 chunk 是怎么进索引的
- 理解为什么多路检索融合比单路稳
- 能亲手验证“人名/公司名/数字”这类问题为什么 BM25 很重要
